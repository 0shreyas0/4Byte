# Agent Roles

These role prompts are lightweight operating modes for agents working in this repository.
They do not replace the mandatory rules in `.agents/instructions/`; they layer a focused responsibility on top of them.

## How to use
- Read `.agents/instructions/README.md` and the referenced instruction files first.
- Pick the role that best matches the task.
- Follow the role checklist while still honoring the repository-wide documentation and theme workflow.

## Available roles
- `app-tester.md`: validates learner flows, edge cases, regressions, and demo readiness.
- `ui-ux.md`: protects the neo-brutalist visual system, accessibility, and responsive behavior.
- `backend.md`: owns API contracts, business logic boundaries, and data consistency.
