---
description: How to populate unique quiz data for all educational stages in NeuralPath
---

To ensure every educational stage has a unique and relevant quiz experience, follow these steps:

1. **Identify the Domain Mappings**: 
   - Open `web/components/edtech/DomainSelection/index.tsx` and note the strings generated by `handleFinalConfirm`.
   - Typically, it's `Subject (Grade)` for school levels or just the `Domain` name for professional tracks.

2. **Prepare Question Sets**:
   - For each subject (e.g., "Physics", "Mathematics", "Accountancy"), prepare a set of at least 8 `QuizQuestion` objects.
   - Each question MUST include:
     - `id`: unique string (e.g., `physics-g11-1`)
     - `topic`: category (e.g., "Mechanics")
     - `concept`: micro-gap tracking (e.g., "vector addition")
     - `question`: the text
     - `options`: 4 strings
     - `correctIndex`: index of the correct answer
     - `explanation`: educational feedback

3. **Update `web/lib/edtech/quizData.ts`**:
   - Add new keys to the `QUIZ_DATA` object matching the mapped strings.
   - Example:
     ```typescript
     "Physics (11th Grade)": [ ... ],
     "Accountancy (11th Grade)": [ ... ],
     ```

4. **Verify Kid-Mode Compatibility**:
   - For "Early Childhood" subjects, ensure `cardStyle: "visual"` is set and `optionVisuals` or `promptVisual` (emojis) are provided.

5. **Test the Flow**:
   - Select a specific stage in the UI and ensure the quiz content correctly reflects the subject instead of defaulting to "DSA".

// turbo
6. **Execute Population**: Use the `multi_replace_file_content` tool to batch-inject the prepared question sets into `web/lib/edtech/quizData.ts`.
